反射率算法处理变化说明

本报告覆盖当前代码中的实际算法逻辑，汇总从最开始的简单比值计算，到后续为信噪比、孤立噪声、ROI 统计、CWL/FWHM 指标所做的完整变化。

代码文件：test/roi_reflectance_eval.py
更新日期：2026-06-22
默认数据路径：F:\05-Jerome Studios\计算光谱专项\测试数据\003
核心原则：关键处理放在除法前的原始差分信号阶段，不在最终曲线上做隐藏或伪修正。
最开始的核心公式： Reflectance = (Sign - Dark) / (Ref - Dark)
当前完整处理流程：
  1. 读取 *-Rec、*-Ref、*-Dark 的 ENVI .hdr/.raw 数据，并检查尺寸与波长轴一致。
  2. 对每个波段先计算 Signal = Sign - Dark、RefSignal = Ref - Dark。
  3. 在原始差分图像上进行空间高斯滤波，默认 spatial_sigma = 0.8。
  4. 如果 Signal < 0 或 RefSignal < 0，该像素反射率置为 0，不进入除法。
  5. 用 dark_std 估计噪声，要求 Signal 与 RefSignal 都大于 max(eps, snr * dark_std)。
  6. 对通过门限的有效像素做连通域过滤，默认至少 8 个相邻像素才保留。
  7. 仅对保留下来的有效像素执行除法；低信噪比或孤立点输出 0，并记录质量字段。
  8. ROI 光谱默认使用 mean(Signal_valid) / mean(RefSignal_valid) 作为均值反射率。
  9. 对每个 ROI 的光谱计算主峰、CWL、FWHM、次峰比例和小峰数量。
当前默认关键参数：
参数 默认值 作用
eps 5.0 Signal/RefSignal 的绝对最低有效门限。
snr 3.0 相对暗噪声标准差的最低信噪比倍数。
spatial_sigma 0.8 除法前对 Signal 和 RefSignal 做空间高斯滤波的强度。
min_valid_fraction 0.05 ROI 中至少需要多少比例像素真正参与除法，才认为该波段质量足够。
min_division_pixels 30 ROI 中实际参与除法的最少像素数。
min_component_pixels 8 通过 SNR 门限的像素必须形成至少 8 个相邻像素的连通区域，孤立点不参与除法。
Auto 5x5 5 x 5，每块面积 30% 自动生成 25 个中心矩形 ROI，并行计算每个 ROI 的光谱与指标。

1. 从直接比值改为先构造 Signal 与 RefSignal

之前

直接把 Sign - Dark 与 Ref - Dark 放进比值公式，缺少中间信号质量判断。

现在

先明确计算 Signal = Sign - Dark 和 RefSignal = Ref - Dark，负数、SNR、连通域等判断都发生在这两个原始差分信号上。

改善：每个像素是否可信可以追溯到原始信号层，而不是只看最终反射率比值。

2. 差分后负数前置归零，不参与除法

之前

如果 Sign - Dark 或 Ref - Dark 出现负数，可能继续进入除法，得到负反射率或异常比值。

现在

只要 Signal < 0 或 RefSignal < 0，该像素输出 0，并记录到 negative_signal_count、negative_ref_signal_count、prezero_count。

改善：避免暗噪声扣除后的负值被除法放大成物理上不合理的结果。

3. 在原始差分图像上加入空间高斯滤波

之前

每个像素直接用原始差分值计算，局部坏点、随机噪声和孤立尖点会影响 ROI 平均。

现在

在除法前，对 Signal 和 RefSignal 做空间高斯滤波，默认 spatial_sigma = 0.8；不对最终光谱曲线做高斯滤波。

改善：从原始图像层降低空间随机噪声，减少孤立坏点对后续除法和 ROI 统计的影响。

4. ROI 计算增加 padding，避免滤波边界误差

之前

如果只截取 ROI 内部再做滤波，ROI 边缘像素会缺少邻域信息，边界区域可能被人为影响。

现在

对 ROI 按 3 * spatial_sigma 自动扩展计算区域，先在扩展区域完成滤波和质量判断，再裁回用户选定的 ROI。

改善：ROI 边缘的信号处理更接近整幅图像计算结果，减少边界效应。

5. 增加基于 Dark 标准差的双 SNR 门限

之前

只要分母 Ref - Dark 大于 0，就可能参与除法；分母很小时，噪声会被放大成假峰。

现在

用 dark_std 估计噪声，要求 Signal 和 RefSignal 同时大于 max(eps, snr * dark_std)，默认 eps = 5、snr = 3。

改善：同时抑制低 Signal 和低 RefSignal 区域，减少小分母和低信噪比造成的异常尖峰。

6. 新增连通域过滤，去掉孤立有效点

之前

有些孤立像素可能刚好超过 SNR 门限，虽然数量很少，但进入除法后会在某些波段形成杂峰。

现在

通过 SNR 门限的像素还必须形成连通区域，默认至少 8 个相邻像素才保留；孤立小块被归入低 SNR，不参与除法。

改善：这是后续为 830 nm 数据前半段杂峰新增的关键算法处理，可以去掉随机孤立点造成的假峰，同时不针对固定波长写死规则。

7. ROI 均值改为 ratio-of-means

之前

更接近逐像素计算 Signal / RefSignal 后再平均，少量异常小分母像素会明显拉高 ROI 结果。

现在

先筛选真正通过门限和连通域过滤的有效像素，再计算 mean(Signal_valid) / mean(RefSignal_valid)。

改善：ROI 光谱均值更稳定，不容易被单个或少数异常像素带歪。

8. 增加有效除法像素数量与比例约束

之前

只要有少量有效像素，就可能输出一个看似正常的 ROI 反射率。

现在

每个 ROI、每个波段都会计算 division_pixel_count 和 division_fraction。默认至少 30 个像素且比例不低于 5%，才被认为质量足够。

改善：防止少量像素支撑整段波长结果，降低偶然噪声造成的假峰风险。

9. 低质量波段保留在完整曲线上，但输出为 0

之前尝试过但已撤销

曾尝试用视觉标记或 mask 来区分异常波段，这会让曲线看起来像被隐藏或被人为修饰。

现在

横坐标完整显示全部波长。如果某个波段在原始差分信号阶段没有足够可靠的有效像素，该波段 mean_reflectance 输出 0，同时 metric_mean_reflectance 为 NaN，并用 quality_status 说明原因。

改善：不遮住波长范围，也不在除法后平滑或裁剪；低质量结果用数据质量逻辑表达为无可靠反射率。

10. CSV 增加完整质量诊断字段

之前

CSV 主要输出反射率结果，难以判断异常来自负值、低 Signal、低 RefSignal、孤立像素还是有效像素不足。

现在

CSV 增加 valid_fraction、division_fraction、quality_status、low_snr_count、low_signal_snr_count、low_ref_snr_count、negative_signal_count、negative_ref_signal_count、prezero_count、dark_std、signal_threshold、ref_threshold、signal_mean、ref_signal_mean、min_component_pixels 等字段。

改善：每个波段都可以复查为什么被保留、为什么归零、为什么不参与指标计算。

11. artifact range 保留为标记，不再改变数据

之前讨论

针对 500 到 600 nm 异常波形，曾考虑过显示层 mask 或特殊范围处理，但这种方式容易丢失信息。

现在

artifact_mode = mark 只写入 artifact_status 标记，不改变 mean_reflectance、不裁剪曲线、不隐藏波段。

改善：异常波长范围仍可追踪，但不再通过固定波长规则修改真实计算结果，避免针对某个主波长或固定区间过拟合。

12. CWL 与 FWHM 改为基于主峰半高点计算

之前

只输出 ROI 曲线，没有直接量化主峰中心波长和半高宽。

现在

先找主峰，再以主峰高度的一半作为半高，左右两侧通过线性插值找半高交点；FWHM = right - left，CWL = (left + right) / 2。

改善：可以定量评价主峰位置和峰宽，而不只是凭肉眼看曲线。

13. FWHM/CWL 增加 fallback，减少无指标 ROI

之前

如果质量曲线 metric_mean_reflectance 里半高交点不足，某些 ROI 会没有 CWL 和 FWHM。

现在

优先使用质量合格曲线计算指标；如果失败原因是找不到半高交点，则回退到完整显示曲线计算。回退成功时记录 metric_status = fallback_raw_half_max。

改善：在不隐藏曲线的前提下提高 CWL/FWHM 的可获得性，同时保留状态字段说明指标来源。

14. 增加次峰与小峰诊断

之前

单色光场景中，只能看到曲线上的小峰，但没有量化它们相对主峰的程度。

现在

输出 secondary_peak_wavelength_nm、secondary_peak_reflectance、secondary_peak_ratio 和 minor_peak_count_10pct。

改善：可以快速判断杂峰是否只是低幅噪声，还是已经达到主峰的显著比例。

15. Auto 5x5 自动 ROI 划分

之前

每次都需要手动画 ROI，ROI 位置和面积容易因人为操作不一致而变化。

现在

将图像平均分成 5 x 5，每个网格中心自动取一个矩形 ROI，面积为该网格面积的 30%。

改善：提供可重复的默认采样方式，便于比较整幅图不同区域的 CWL/FWHM 和反射率差异。

16. Auto 5x5 计算改为后台并行

之前

自动生成 25 个 ROI 后，在 GUI 主线程逐个计算，窗口容易卡顿。

现在

使用 ThreadPoolExecutor 后台计算，默认最多 4 个 worker，并通过定时器轮询结果，完成后一次性绘制 ROI 和曲线。

改善：Auto 5x5 的交互卡顿明显减少，ROI 结果仍按同一套算法计算。

17. 图像预览 Colorbar 使用全局一致范围

之前

切换不同波段时，Colorbar 可能随当前图像自动缩放，视觉亮度不便横向比较。

现在

从多个波段抽样估计全局预览范围，切换 band 时保持统一 vmin/vmax。

改善：图像预览更适合跨波段比较。注意这只影响显示，不改变反射率计算和导出的 CSV 数值。

18. 完整曲线显示，不针对随机主波长写死规则

之前风险

如果针对 500 到 600 nm 或某个特定波长范围做特殊处理，可能在换成其他主波长数据时失效或误伤有效信号。

现在

算法不根据固定波长区间删除或平滑数据。主峰位置由数据本身决定，所有波长点都在曲线上显示；异常点通过原始信号 SNR、连通域和有效像素数量判断处理。

改善：更适合主波长随机变化的输入数据，不会为了某一组数据而过拟合固定波段。
整体结论： 当前算法已经从最初的简单 (Sign - Dark) / (Ref - Dark)，升级为“原始差分信号降噪、负值前置归零、Signal/RefSignal 双 SNR 门限、连通域孤立点过滤、ratio-of-means ROI 统计、低质量波段质量标记、CWL/FWHM 与次峰诊断”的完整流程。核心目标不是让曲线看起来更平滑，而是从原始数据层减少低信噪比、小分母、孤立坏点造成的假峰，同时保持全波长曲线可见和质量字段可追溯。

说明：GUI 布局、中文字体、Message Box 等界面优化未作为算法变化展开；它们不改变反射率、CWL 或 FWHM 的计算结果。
